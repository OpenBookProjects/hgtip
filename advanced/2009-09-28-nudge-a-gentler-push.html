Nudge — 温和的推送

posted by Steve Losh on September 28, 2009

令 git 用户困惑的一件事儿是 Mercurial 在执行 hg push 时,实际上是尝试将整个仓库的变更都进行推送, 而 git 仅将当前分支的变更进行推送.

如果你想保持 git 的习惯(也许你经常工作在永远也不会推送的分支中), 这有个简单的操作别名可以作到.

Mercurial 支持你使用 . 来指代”当前工作目录的父版本.” 在推送时使用就可以使用 hg push --rev . 来仅将当前工作目录的变更推送上去(包含继承的变更). 这样,你就不用担心将其它分支的变更意外的推送上去了.

编辑你的 ~/.hgrc  增补这样的别名:

[alias]
nudge = push --rev .

现在你就可以使用 hg nudge 来仅将刚刚的修订上推了.
